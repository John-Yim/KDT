📏 정규성 검정¶

본 분석에서는 회귀 및 상관분석에 앞서, 주요 변수들이 정규분포를 따르는지를 검정함.
정규성을 만족하지 않을 경우, 비모수 분석 방법을 사용해야 함.

✅ 검정 방법: Shapiro-Wilk Test¶

  • Shapiro-Wilk Test는 정규성을 검정하는 대표적인 통계적 방법으로,
    p-value < 0.05일 경우 정규성을 만족하지 않는다고 판단함.
  • 분석 대상 변수:
    • btc_price, eth_price, doge_price (암호화폐 가격)
    • kospi, nasdaq (지수)
    • kor_rate, usa_rate (금리)
    • moon_phase, weather (달의 위상, 날씨)

from scipy.stats import shapiro

cols = ['btc_price', 'eth_price', 'doge_price', 'kospi', 'nasdaq', 'kor_rate', 'usa_rate', 'moon_phase', 'partly_cloudy', 'snow', 'clear', 'rain', 'cloudy']

for col in cols:
    stat, p = shapiro(df[col].dropna())
    print(f"{col} → p-value: {p:.4f} → {'✅ 정규분포' if p > 0.05 else '❌ 비정규'}")

📊 상관관계 분석 개요¶

  • 데이터 분포가 정규성을 만족하지 않기 때문에,
    비모수 분석을 적용함

  • 이에 따라 스피어만 상관계수 를 사용하여
    암호화폐 가격과 경제 지표 간의 순위 기반 상관관계를 분석함

✅ 분석 대상 변수¶

  • 종속 변수:
    btc_price, eth_price, doge_price

  • 독립 변수:
    kospi, nasdaq, kor_rate, usa_rate, moon_phase, partly_cloudy, snow, clear, rain, cloudy

✅ 분석 기준¶

  • 비모수 상관계수: rho (ρ)
  • 유의수준 (α): 0.05
    → p-value < 0.05인 경우 통계적으로 유의한 상관관계로 해석

📌 Spearman 상관계수는 데이터의 값 자체가 아닌 순위(랭크)를 기반으로 계산되며,
정규성을 만족하지 않는 데이터 간 관계 분석에 적합함.

from scipy.stats import spearmanr

print("📊 스피어만 상관관계 분석 결과:\n")

for target in ['btc_price', 'eth_price', 'doge_price']:
    print(f"▶ {target} 와의 상관계수:")
    for feature in ['kospi', 'nasdaq', 'kor_rate', 'usa_rate', 'moon_phase', 'partly_cloudy', 'snow', 'clear', 'rain', 'cloudy']:
        coef, p = spearmanr(df[target], df[feature])
        print(f"   {feature:10} → ρ = {coef:.4f} (p = {p:.4f}) {'✅ 유의' if p < 0.05 else '⚠️ 유의X'}")
    print()

📊 상관관계 분석 결론¶

🔬 가설 1: 경제지표와 암호화폐 가격 간의 관계¶

  • 모든 분석 결과에서 p-value < 0.05를 만족한 변수들이 다수 존재하였으며, 이에 따라
    귀무가설(H₀₁): "암호화폐 가격은 경제 지표와 상관관계가 없다" 는 기각 → 대립가설(H₁₁): "경제 지표와 상관관계가 있다"를 채택

🔍 주요 해석 포인트¶

  • 나스닥 지수와의 상관관계는 매우 강하게 나타남
    → 이는 암호화폐 시장이 글로벌 증시 흐름과 밀접하게 연동되어 있음을 시사함

  • 코스피와는 중간 정도의 양의 상관관계를 가짐

  • 한국 금리와는 명확한 음의 상관관계
    → 금리가 상승하면 암호화폐 가격은 하락하는 경향

  • 미국 금리와는 매우 약한 음의 상관관계를 보임
    → 이는 분석에 사용된 업비트 원화 마켓 기준의 가격 데이터 특성 때문으로 판단됨

📌 추가 분석 제안¶

📝 보다 정밀한 해석을 위해서는 달러 마켓(Binance 등) 데이터와의 비교 분석이 필요함

  • 업비트(원화 마켓) vs 바이낸스(달러 마켓) 간 가격 흐름 비교
  • 미국 금리의 영향을 더 직접적으로 반영하는 USD 기반 가격 데이터 분석 고려

🔬 가설 2: 비정형 요인과 암호화폐 가격 간의 관계¶

  • 달의 위상 및 날씨 조건(맑음, 흐림 등) 과의 관계는 대부분 p-value > 0.05로 나타났으며,
    이에 따라 대립가설(H₁₂): "비정형 요인과 상관관계가 있다" 는 기각 → 귀무가설(H₀₂): "비정형 요인과 상관관계가 없다"를 채택

🔍 주요 해석 포인트¶

  • 날씨 요인은 대부분 통계적으로 유의하지 않았음
    → 예외적으로 DOGE의 경우 맑음, 눈, 비에서 약한 상관관계가 관찰됨

  • 달의 위상 은 세 코인 모두에서 유의하지 않은 결과
    → 암호화폐 가격과의 직접적인 관계는 확인되지 않음

🧠 종합 결론¶

  • 암호화폐 가격은 전통적인 경제 지표(나스닥, 금리, 코스피 등)와 통계적으로 유의한 상관관계를 가짐
  • 반면, 날씨 및 달의 위상 등 비정형 변수는 유의미한 영향력이 없는 것으로 판단됨



📈 선형 관계 분석 (Linear Relationship Analysis)¶

앞서 수행한 상관관계 분석 결과를 바탕으로 암호화폐 가격에 영향을 미치는 다양한 외부 변수들을
보다 구체적으로 분석하고자 다중 선형 회귀분석 을 수행. 비정형 요인 중 통계적으로 유의한 경향을 보였던 도지코인(DOGE)과 날씨 변수 간의 관계도 추가적으로 분석.

✅ 분석 목적¶

  • 경제 지표(나스닥, 코스피, 금리 등)가 암호화폐 가격에 미치는 영향력을 수치적으로 파악
  • 독립 변수들의 조합이 종속 변수(암호화폐 가격)를 얼마나 잘 설명하는지(R²) 평가
  • 비정형 요인인 날씨가 도지코인 가격에 실질적인 영향을 미치는지 회귀 분석을 통해 확인

✅ 분석 방식¶

🔹 경제지표 기반 회귀분석¶

  • 종속 변수: btc_price, eth_price, doge_price (각각 개별 회귀 모델 구성)
  • 독립 변수: kospi, nasdaq, kor_rate, usa_rate
  • 도구: statsmodels 라이브러리의 ols() 사용

🔹 도지코인-날씨 회귀분석¶

  • 종속 변수: doge_price
  • 독립 변수: clear, cloudy, rain, snow, partly_cloudy (날씨 상태, one-hot 인코딩)
  • 전처리: 날씨 변수는 0/1 값의 더미 변수로 사용 (추가 정규화 불필요)
  • 도구: statsmodels 라이브러리의 ols() 사용

⏱️ 시계열 특성을 고려한 상관 분석¶

앞선 상관관계 분석은 시계열 데이터를 단순 수치형 변수로 처리하여, 시간 흐름에 따른 영향은 고려하지 않은 정태적 분석이었다.
그러나 암호화폐 가격과 외부 요인은 모두 시계열적 특성을 가지므로 시간의 흐름에 따른 관계성을 분석하는 것이 더 현실적이고 타당하다.

✅ 1. 시차 상관관계 분석 (Cross-Correlation Function)¶

  • 암호화폐 가격과 외부 요인 간에 며칠 후에 영향을 주고받는가? 를 파악하기 위해 CCF 분석을 수행하였다.
  • 각 변수 쌍에 대해 lag 0~29일 범위의 시차를 적용해 상관계수를 계산하였고 가장 강한 상관을 보인 lag 값을 기록하였다.
In [ ]:
import numpy as np
import pandas as pd
from statsmodels.tsa.stattools import ccf
import matplotlib.pyplot as plt
import seaborn as sns

df_norm = pd.read_csv['암호화폐_금리_지수_월령_날씨_병합데이터_정규화.csv']

# 설정
crypto_cols = ['btc_price', 'eth_price', 'doge_price']
external_cols = ['kospi', 'nasdaq', 'kor_rate', 'usa_rate', 'moon_phase',
                 'partly_cloudy', 'snow', 'clear', 'rain', 'cloudy']
max_lag = 30
summary_data = []

# CCF 계산
for coin in crypto_cols:
    for external in external_cols:
        series1 = df_norm[coin]
        series2 = df_norm[external]
        min_len = min(len(series1), len(series2))
        series1 = series1[:min_len]
        series2 = series2[:min_len]

        # CCF 계산 (lag 0 ~ 29)
        corr_values = ccf(series1, series2)[:max_lag]
        max_corr = np.max(np.abs(corr_values))
        best_lag = np.argmax(np.abs(corr_values))

        summary_data.append({
            'coin': coin,
            'external_variable': external,
            'max_abs_correlation': max_corr,
            'best_lag': best_lag,
            'annot': f"{max_corr:.2f}\n(lag={best_lag})"
        })

# DataFrame 변환
ccf_summary_df = pd.DataFrame(summary_data)

# 피벗 테이블 생성
heatmap_values = ccf_summary_df.pivot(index='coin', columns='external_variable', values='max_abs_correlation')
heatmap_annots = ccf_summary_df.pivot(index='coin', columns='external_variable', values='annot')

# 시각화
plt.figure(figsize=(12, 6))
sns.heatmap(heatmap_values, annot=heatmap_annots, fmt='', cmap='coolwarm', cbar=True)
plt.title("Coin vs External Variable - Max Abs Correlation + Lag")
plt.ylabel("Cryptocurrency")
plt.xlabel("External Variable")
plt.tight_layout()
plt.show()

✅ 2. 공적분 분석 (Cointegration Test)¶

  • 비정상 시계열인 암호화폐 가격과 외부 요인 간에 장기적인 균형 관계가 존재하는지를 검증
  • Engle-Granger 공적분 검정을 통해 3개 코인 × 외부 요인 전체 조합에 대해 p-value를 산출

import pandas as pd
import numpy as np
from statsmodels.tsa.stattools import coint

df = pd.read_csv('암호화폐_금리_지수_월령_날씨_병합데이터.csv')

crypto_cols = ['btc_price', 'eth_price', 'doge_price']
external_cols = ['kospi', 'nasdaq', 'kor_rate', 'usa_rate', 'moon_phase',
                 'partly_cloudy', 'snow', 'clear', 'rain', 'cloudy']
cointegration_results = []

for coin in crypto_cols:
    for external in external_cols:
        y1 = df[coin]
        y2 = df[external]
        min_len = min(len(y1), len(y2))
        y1 = y1[:min_len]
        y2 = y2[:min_len]

        coint_stat, p_value, _ = coint(y1, y2)

        cointegration_results.append({
            'coin': coin,
            'external_variable': external,
            'coint_statistic': coint_stat,
            'p_value': p_value
        })

coint_df = pd.DataFrame(cointegration_results)
coint_df_sorted = coint_df.sort_values(by='p_value')

# 5. 출력
print(coint_df_sorted)
